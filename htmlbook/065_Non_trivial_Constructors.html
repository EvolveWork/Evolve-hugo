Initialization is a significant stumbling block. Your code might do all the right things, but if you don’t set up the proper initial conditions it won’t work correctly.

Each object is its own isolated world. A program is a collection of objects, so correct initialization of each object solves a large part of the initialization problem. Kotlin provides mechanisms to guarantee proper object initialization.
